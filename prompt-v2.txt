В проекте уже существует Telegram-бот в каталоге @bot , работающий 
через Haystack и кастомный API.

Задача — разработать вторую версию бота в отдельной директории 
с модульной архитектурой, полной декомпозицией и разделением логики 
по файлам.

Функционально новая версия должна полностью повторять текущие 
возможности бота + добавить полноценную обработку 
файлов (PDF, DOCX и т.д.), их анализ через DocLing, и сохранение 
структурированного контента в векторную базу.

1. Цель:

Собрать новую, масштабируемую архитектуру Telegram-бота, 
в которой компонентный подход и Haystack-пайплайны являются ядром. 
Бот должен:
    - полностью повторять существующие функции текущей версии;
    - добавлять работу с файлами через DocLing;
    - сохранять контент файлов в векторную базу Chroma;
    - автоматически выдавать пользователю краткое резюме 
    содержимого файла после успешной обработки;
    - учитывать документы при последующих запросах, используя 
    retriever и пайплайн генерации ответа.

2. Архитектура проекта (папки и файлы):

Создать отдельную директорию, например, bot_v2.
Структура каталогов внутри:
    components (модули компонентов)
    pipelines (модули пайплайна)
    bot (модули бота)

Модули новой версии бота не должны зависеть от модулей страой версии.
Каталог локальных данных должен быть новым - @data_v2 .

3. Общая логика бот-процесса

3.1. Обработка текстовых сообщений

- Пользователь пишет любое сообщение.
- Активируется generation_pipeline:
- Retriever ищет релевантные куски сохранённых документов.
- LLM формирует ответ, учитывая найденные данные.
- Бот отправляет пользователю результат.

3.2. Обработка файлов

Если пользователь присылает документ любого поддерживаемого формата:
- Бот отправляет сообщение: «Файл получен. Запускаю анализ и сохранение. 
Это может занять немного времени…»
- Файл скачивается и передаётся в DocLing.
- Запускается ingestion_pipeline:
- DocLing анализирует файл и разбивает его на страницы / чанки.
- Каждый чанк отправляется в Chroma. В метаданные обязательно записать:
    - название файла,
    - номер чанка или страницы (если доступно).

После завершения загрузки документов:
- Бот отправляет сообщение: «Готово. Я изучил этот файл, теперь можем 
его обсудить.»
- Далее бот отправляет одно короткое резюме по содержимому 
файла — ровно одно предложение.

    Функциональные требования

4. Повтор текущего функционала

Все существующие функции старой версии бота должны работать без изменений:
обработка текстов, вызов API, настройка пайплайнов, отправка сообщений.
